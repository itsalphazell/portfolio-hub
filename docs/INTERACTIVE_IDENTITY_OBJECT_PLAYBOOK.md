# Interactive Identity Object Playbook

Use this pack when the premium page needs one object that carries brand memory by itself.

## Best fit
- premium-immersive hero with one signature object
- launch pages with one product or symbolic object
- showcase sections where the object is part of the narrative, not just decoration

## When to use
- the object clarifies the brand or product world
- the scene can still preserve CTA readability
- the fallback can preserve the same idea without runtime dependency

## When not to use
- no clear object concept exists
- the page is already visually dense
- the object is only there to create generic wow
- product clarity is already weak without it

## Execution rules
1. One object, one story, one focal region.
2. Start with the still frame before pointer motion.
3. Keep interaction light: hover, pointer drift, slow orbit, or one intentional state change.
4. Never let the object mask the CTA or the value proposition.
5. Ship a static or simplified fallback for compact viewports and reduced motion.

## QA gates
- hero still reads when the object is present
- object remains secondary to the main CTA
- fallback keeps the same brand idea
- no layout shift or focus issue is caused by the object layer
