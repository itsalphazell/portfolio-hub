# codex-premium-motion

Reusable premium motion building blocks.

- `react/` contains React and GSAP helpers.
- `static/` contains HTML/CSS/JS examples for static sites.
- Keep these assets isolated and copy only what the page needs.
- Start from `react/presets/EditorialShowcaseHero.tsx` when the target is an AWWWARDS-lite premium portfolio or agency landing page.
- Start from `react/presets/ProductBlurSlider.tsx` when the target is a product or campaign page that needs one premium spotlight slider instead of a generic carousel.
- Start from `react/presets/CyberTerminalHero.tsx` when the target is a technical portfolio or security-facing hero that needs a stronger terminal-led identity without escalating into a full scene.
- Use `react/components/PerspectiveTiltHeadline.tsx` when the page needs more tactile type depth before escalating to immersive structure.
- Use `react/components/PointerMediaReveal.tsx` when one visual should react to pointer intent without becoming a full immersive stage.
- Use `react/components/CardShowcaseStack.tsx` when a section needs premium depth from cards instead of from a scene.
- Use `react/components/AchievementCarousel.tsx` when proof, awards, or certifications need stronger pacing than a static badge grid.
- Start from `static/html/editorial-showcase-hero.html` when the target is a static premium-web page with the same editorial pattern.
- Start from `static/html/product-blur-slider.html` for static campaign and product pages.
- Start from `static/html/cyber-terminal-hero.html` for static technical or cyber premium hero surfaces.
- Start from `static/html/pointer-media-reveal.html` for hover-driven media reveals on static pages.
- Start from `static/html/card-showcase-stack.html` for static premium card sections.
- Start from `static/html/achievement-carousel.html` for static trust and milestone sections.
